# Accessibility

WeiUI is built with accessibility as a first-class concern. The system targets WCAG 2.2 AAA for content text, enforces AA as a hard minimum for interactive elements, and ships sensible defaults for keyboard navigation, screen readers, and motion preferences.

## Contrast Standards

| Context | Standard | Ratio |
|---------|----------|-------|
| Body text, headings | WCAG AAA | ≥ 7:1 |
| Large text (18pt+) | WCAG AAA | ≥ 4.5:1 |
| Interactive controls, accent colors | WCAG AA | ≥ 4.5:1 |
| Decorative, disabled states | None | — |

The token system is designed so semantic color pairings meet these ratios in both light and dark mode. Override tokens carefully — introducing a custom `--wui-color-primary` may break contrast guarantees.

## Touch Targets

All interactive elements meet the WCAG 2.2 Success Criterion 2.5.8 (Target Size) minimum of 24×24 CSS pixels. Buttons and form controls default to a minimum height of 44px to align with Apple and Google platform guidelines.

```css
/* Built into every interactive component */
.wui-button {
  min-height: 2.75rem; /* 44px */
  min-width: 2.75rem;
}
```

## Focus Indicators

Every focusable element has a visible, high-contrast focus ring. WeiUI uses a `2px` offset outline that inherits the primary color to stay theme-aware.

```css
/* Applied globally via the CSS layer */
:focus-visible {
  outline: 2px solid var(--wui-color-ring);
  outline-offset: 2px;
}
```

The `:focus-visible` pseudo-class is used rather than `:focus` so the ring only appears during keyboard navigation, not on mouse click.

## Keyboard Navigation

WeiUI components follow the [ARIA Authoring Practices Guide](https://www.w3.org/WAI/ARIA/apg/) keyboard patterns:

| Component | Keys |
|-----------|------|
| Button | `Enter`, `Space` to activate |
| Tabs | `Arrow Left/Right` to switch, `Home/End` for first/last |
| Dialog | `Escape` to close, focus trapped inside while open |
| Menu | `Arrow Up/Down` to navigate, `Enter` to select, `Escape` to close |
| Select | `Arrow Up/Down` to navigate options |
| Checkbox | `Space` to toggle |
| Radio group | `Arrow Up/Down/Left/Right` to change selection |

## Screen Reader Support

Components ship with the appropriate ARIA roles, states, and properties. You do not need to add `role` or `aria-*` attributes manually for standard usage.

```html
<!-- Role and state managed automatically -->
<button class="wui-button wui-button--solid" aria-busy="true">
  <span class="wui-spinner" aria-hidden="true"></span>
  Loading...
</button>

<!-- Required field announced correctly -->
<label for="email">Email <span aria-hidden="true">*</span></label>
<input id="email" class="wui-input" type="email"
       aria-required="true" aria-describedby="email-error" />
<span id="email-error" class="wui-field-error" role="alert">
  Please enter a valid email address.
</span>
```

### Live-region announcer

For imperative announcements — copy-to-clipboard success, async save confirmations, form submissions — use the `announce` helper from `@weiui/headless`. It creates (once) a visually-hidden live region and writes your message into it, so assistive technology reads it aloud without moving focus.

```tsx
import { announce } from "@weiui/headless";

async function onCopy(value: string) {
  await navigator.clipboard.writeText(value);
  announce("Copied to clipboard");
}

// Use "assertive" for urgent errors that should interrupt the current read-out.
announce("Save failed — try again", "assertive");
```

The first call mounts a `<div role="status" aria-live="polite">` (or `role="alert" aria-live="assertive"`) on `document.body`. Subsequent calls reuse the same node so screen readers stay in sync with repeated updates.

## Motion Preferences

All animations and transitions respect `prefers-reduced-motion`. When the user has enabled reduced motion in their OS settings, transitions are shortened or removed entirely.

```css
/* Built into the WeiUI CSS layer */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

## Semantic HTML

The CSS layer is designed to augment semantic HTML — not replace it. Use the right element for the job and add a WeiUI class for styling:

```html
<!-- Good: semantic element + WeiUI class -->
<button class="wui-button wui-button--solid">Submit</button>
<nav class="wui-nav" aria-label="Main navigation">...</nav>
<main class="wui-main">...</main>

<!-- Avoid: non-semantic element -->
<div class="wui-button" onclick="...">Submit</div>
```

## Testing Checklist

Before shipping a component or page, verify:

- [ ] All text meets contrast requirements (use browser DevTools or [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/))
- [ ] All interactive elements are reachable and operable by keyboard alone
- [ ] Focus is visible at all times during keyboard navigation
- [ ] Screen reader announces correct role, state, and name for each interactive element
- [ ] Animations are suppressed under `prefers-reduced-motion: reduce`
- [ ] Touch targets are at least 24×24px (44×44px recommended)
- [ ] Form fields have associated labels and error messages linked via `aria-describedby`

## Per-component testing checklist

Before shipping a feature that uses any of these components, verify
the listed behaviors by hand (or via automated axe-core / Playwright
tests). Each box represents the AAA minimum — fix, don't suppress.

### Dialog
- [ ] Focus moves into the dialog on open; first focusable element receives it
- [ ] Focus trap — Tab cycles inside only
- [ ] Escape closes; overlay click closes unless modal is prevent-closeable
- [ ] `role="dialog"` + `aria-labelledby` + `aria-describedby` wired
- [ ] Background content is `aria-hidden="true"` while open
- [ ] Focus returns to the trigger on close

### Drawer
- [ ] Same focus-trap + Escape + background aria-hidden as Dialog
- [ ] `aria-label` or `aria-labelledby` on the drawer content
- [ ] Swipe-to-dismiss gesture doesn't steal keyboard focus
- [ ] `onInteractOutside` cancelable when critical work is unsaved

### Popover
- [ ] Focus moves to first focusable on open (when `modal` is true)
- [ ] Escape closes
- [ ] Outside click closes unless prevented
- [ ] Placement flips when near viewport edge
- [ ] Trigger has `aria-expanded` reflecting state

### Menu
- [ ] Arrow Up / Down / Home / End navigate items
- [ ] First-letter typeahead focuses the matching item
- [ ] Enter / Space invokes selected item
- [ ] Escape closes and returns focus to trigger
- [ ] `role="menu"` on content, `role="menuitem"` on each item
- [ ] Disabled items skipped by arrow nav

### Tooltip
- [ ] Shows on focus AND on hover
- [ ] Dismissible with Escape
- [ ] `role="tooltip"` + `aria-describedby` on trigger when open
- [ ] Doesn't cover the trigger; doesn't flicker

### Toast
- [ ] `role="status"` (info) or `role="alert"` (error / success)
- [ ] `aria-live="polite"` for default, `assertive` for errors
- [ ] Auto-dismiss pausable on hover / focus
- [ ] Swipe-to-dismiss doesn't conflict with scroll gesture
- [ ] Screen reader announces the message once

### CommandPalette
- [ ] Cmd/Ctrl+K opens; Escape closes
- [ ] Arrow Up / Down moves selection; Enter invokes
- [ ] Input has `aria-controls` pointing to the results list
- [ ] Each item `role="option"` inside a `role="listbox"`
- [ ] `aria-live="polite"` announces empty-state / result counts

### Tabs
- [ ] `role="tablist"` / `tab` / `tabpanel` wired
- [ ] Arrow keys (Left/Right for horizontal, Up/Down for vertical) cycle
- [ ] Home / End jump to first/last
- [ ] `aria-controls` and `aria-selected` correct per tab
- [ ] Activation mode (auto vs manual) respected
- [ ] Tab key focuses current panel, not next tab

### Accordion
- [ ] ArrowUp / Down cycles between triggers
- [ ] Home / End jump to first / last trigger
- [ ] Enter / Space toggles the expanded state
- [ ] `aria-expanded` + `aria-controls` wired on trigger
- [ ] Content uses `role="region"` with `aria-labelledby` = trigger id
- [ ] Animated expand keeps content in DOM (inert when closed)

### Select / AutoComplete / MultiSelect
- [ ] Arrow Down opens the popup when focused
- [ ] Typeahead or async filtering announced
- [ ] Enter selects; Escape closes
- [ ] `role="combobox"` + `aria-controls` + `aria-expanded` on input
- [ ] Selected option has `aria-selected="true"`
- [ ] MultiSelect: Backspace removes last tag

### RadioGroup / Checkbox / Switch
- [ ] Label clicks toggle the control
- [ ] `aria-invalid` set when inside an errored `<Field>`
- [ ] RadioGroup: Arrow keys cycle; roving tabindex
- [ ] Switch has `role="switch"` + `aria-checked`
- [ ] Disabled state communicated via both `aria-disabled` and `:disabled`

### ToggleGroup
- [ ] Roving tabindex
- [ ] Arrow keys cycle, Home/End jump
- [ ] `role="group"` with `aria-label` or `aria-labelledby`
- [ ] Each item `role` depends on `type`: radio (single) / button (multiple)

### Slider
- [ ] Arrow keys adjust; PageUp / PageDown jumps by 10 steps
- [ ] Home / End jump to min / max
- [ ] `aria-valuemin` / `max` / `now` / `valuetext` wired
- [ ] Vertical orientation respects Up/Down correctly

### DatePicker / Calendar
- [ ] Arrow keys move day-by-day
- [ ] PageUp / Down cycle months; Shift+Page cycles years
- [ ] Home / End jump to start / end of week
- [ ] Enter selects; Escape closes the popover
- [ ] `aria-current="date"` on today; `aria-selected` on chosen
- [ ] Disabled dates skipped during keyboard nav

### Stepper
- [ ] Each step `role="listitem"` inside `role="list"` (or `<ol>`)
- [ ] `aria-current="step"` on active
- [ ] Errored step announced via `aria-live`
- [ ] Clickable steps have `<button>` (not just `<div>`)

### TreeView
- [ ] `role="tree"` / `treeitem` / `group`
- [ ] Arrow keys navigate; Right expands, Left collapses
- [ ] First-letter typeahead focuses the matching node
- [ ] Roving tabindex (only one tab-stop)
- [ ] `aria-expanded` on branch items

### DataTable
- [ ] `role="grid"` when keyboard-navigable cells
- [ ] Arrow keys move between cells
- [ ] Sort direction announced via `aria-sort`
- [ ] Selection column uses `<input type="checkbox">` with `aria-label`
- [ ] Sticky header scrolls horizontally with body

### Editor
- [ ] Toolbar buttons have labels + `aria-pressed` for toggle formatting
- [ ] Focus ring on the contenteditable surface
- [ ] Shortcut hints in tooltip titles (`Bold (Cmd+B)`)
- [ ] Screen reader can enter/exit the contenteditable normally

### InputOTP
- [ ] Each segment `aria-label="Digit 1 of 6"` (incrementing)
- [ ] Arrow Left/Right moves between segments
- [ ] Auto-advance on digit entry
- [ ] Backspace on empty segment focuses previous

### ColorPicker
- [ ] Alpha slider + hue slider operate as `role="slider"` with aria values
- [ ] Hex input focusable + editable
- [ ] Announce color change via `aria-live`
- [ ] Disabled state rendered
