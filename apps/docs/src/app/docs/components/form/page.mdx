### Indeterminate

A tri-state "some children selected" mode commonly used in select-all lists. Set `indeterminate` and the checkbox renders a dash glyph and exposes `aria-checked="mixed"`.

```tsx
import { useState } from "react";
import { Checkbox } from "@weiui/react";

function SelectAll({ items, selected, setSelected }) {
  const allChecked = selected.length === items.length;
  const someChecked = selected.length > 0 && !allChecked;

  return (
    <Checkbox
      label="Select all"
      checked={allChecked}
      indeterminate={someChecked}
      onChange={(e) => setSelected(e.target.checked ? items : [])}
    />
  );
}
```

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| label | `string` | — | Visible label text |
| checked | `boolean` | — | Controlled checked state |
| defaultChecked | `boolean` | — | Initial uncontrolled checked state |
| indeterminate | `boolean` | `false` | Tri-state mixed indicator (aria-checked="mixed") |
| invalid | `boolean` | — | Shows the invalid style + sets `aria-invalid`. Inherits from `<Field error>` if not set. |
| size | `"sm" \| "md" \| "lg"` | `"md"` | Visual size |
| disabled | `boolean` | `false` | Disables the checkbox |

### Accessibility

- Label is linked to the input via `htmlFor` / `id`
- Keyboard: Space to toggle
- `indeterminate` exposes `aria-checked="mixed"` to assistive tech
- Supports `aria-label`, `aria-describedby`

```tsx
import { useState } from "react";
import { Switch } from "@weiui/react";

// Uncontrolled
<Switch label="Dark mode" defaultChecked />

// Controlled
const [enabled, setEnabled] = useState(false);

<Switch
  label="Notifications"
  checked={enabled}
  onChange={(e) => setEnabled(e.target.checked)}
/>
```

Switch is a native checkbox under the hood with `role="switch"`, so React's usual controlled/uncontrolled rules apply: pass `checked` + `onChange` for controlled, or `defaultChecked` for uncontrolled.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| label | `string` | — | Visible label text |
| checked | `boolean` | — | Controlled checked state |
| defaultChecked | `boolean` | — | Initial uncontrolled checked state |
| onChange | `(e) => void` | — | Change handler (required when controlled) |
| invalid | `boolean` | — | Shows the invalid style + sets `aria-invalid`. Inherits from `<Field error>` if not set. |
| size | `"sm" \| "md" \| "lg"` | `"md"` | Visual size |
| disabled | `boolean` | `false` | Disables the switch |

### Accessibility

- Uses `role="switch"` so screen readers announce it correctly
- `aria-checked` is set automatically from the checked state
- Keyboard: Space to toggle

```tsx
  <RadioGroupItem value="a" label="Apple" />
  <RadioGroupItem value="b" label="Banana" />
  <RadioGroupItem value="c" label="Cherry" />

```

### RadioGroup Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| value | `string` | — | Controlled selected value |
| defaultValue | `string` | `""` | Initial uncontrolled value |
| onValueChange | `(value: string) => void` | — | Called when selection changes |

### Keyboard

- ArrowUp / ArrowDown (and ArrowLeft / ArrowRight) move focus between items
- Home / End jump to the first / last item
- Focus follows selection: moving focus selects the target and fires `onValueChange`
- Only the selected item (or the first item if none is selected) is tabbable — this is a roving tabindex
- Disabled items are skipped

### Accessibility

- Container has `role="radiogroup"`
- Items use native `<input type="radio">` with shared `name`, exposing `aria-checked` automatically
- Arrow-key roving with focus-follows-selection, matching WAI-ARIA Authoring Practices

## Field

`Field` wires a label, description, error message, and input together — it auto-generates an `id`, sets `htmlFor` on the label, builds `aria-describedby` from the description + error, and toggles `aria-invalid` when an error is present. Consumers don't need to manage any of that themselves.

```tsx
import {
  Field,
  FieldLabel,
  FieldDescription,
  Input,
} from "@weiui/react";

// All ARIA wiring happens automatically
<Field>
  <FieldLabel>Email</FieldLabel>
  <Input type="email" placeholder="you@example.com" />
  <FieldDescription>We'll only use this to send receipts.</FieldDescription>
</Field>

// With an error — aria-invalid + aria-describedby update automatically
<Field error="Please enter a valid email address">
  <FieldLabel>Email</FieldLabel>
  <Input type="email" defaultValue="not-an-email" />
</Field>
```

The child input can be any component that reads the `FieldContext` — `Input`, `Textarea`, `Select`, and the other form controls in `@weiui/react` all support it out of the box.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| error | `string` | — | Renders an inline error message and wires `aria-invalid` on the child input |
| required | `boolean` | `false` | Adds a required-marker asterisk to `<FieldLabel>` |
| children | `ReactNode` | — | `FieldLabel`, input, and `FieldDescription` in any order |

### Accessibility

- `id` auto-generated via `useId()` and applied to the child input
- `FieldLabel` gets a matching `htmlFor`
- `FieldDescription` gets an id which is combined into `aria-describedby`
- `error` renders an inline error region and sets `aria-invalid="true"` on the input
- Works with screen readers out of the box — no manual ARIA plumbing required
