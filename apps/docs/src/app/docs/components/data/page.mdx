## DataTable

Powered by `@tanstack/react-table` ‚Äî sort, filter, paginate, and select rows.

  code={`import { DataTable, type ColumnDef } from "@weiui/react/data-table";

type User = { name: string; email: string; role: string };

const columns: ColumnDef<User>[] = [
  { accessorKey: "name", header: "Name" },
  { accessorKey: "email", header: "Email" },
  { accessorKey: "role", header: "Role" },
];

<DataTable data={users} columns={columns} searchable pageSize={5} />`}
>
  <DataTableDemo />

```tsx
import { DataTable, createColumnHelper } from "@weiui/react/data-table";

type User = { name: string; email: string; role: string };

const columnHelper = createColumnHelper<User>();
const columns = [
  columnHelper.accessor("name", { header: "Name" }),
  columnHelper.accessor("email", { header: "Email" }),
  columnHelper.accessor("role", { header: "Role" }),
];

<DataTable data={users} columns={columns} searchable pageSize={10} />
```

### Row selection

Pass `selectable` to render a checkbox column plus a header checkbox that selects/deselects the current page. Subscribe to changes with `onRowSelectionChange` ‚Äî it follows the TanStack Table `OnChangeFn<RowSelectionState>` shape so you can forward it straight through.

```tsx
import type { RowSelectionState } from "@tanstack/react-table";

const [selection, setSelection] = useState<RowSelectionState>({});

<DataTable
  data={users}
  columns={columns}
  selectable
  onRowSelectionChange={setSelection}
/>
```

### Loading state

When `loading` is set, rows are replaced with skeleton shimmer. Use this while fetching so the table keeps its layout instead of collapsing.

```tsx
<DataTable data={[]} columns={columns} loading />
```

### Page-size selector

`pageSizeOptions` renders a dropdown in the pagination bar so users can pick how many rows to show.

```tsx
<DataTable
  data={users}
  columns={columns}
  pageSize={10}
  pageSizeOptions={[10, 25, 50, 100]}
/>
```

### Server-side data

For datasets that live on the server, opt-out of client-side pagination / sorting / filtering with the `manual*` flags, supply `rowCount`, and respond to state change callbacks by fetching the next page yourself.

```tsx
<DataTable
  data={pageRows}
  columns={columns}
  manualPagination
  manualSorting
  manualFiltering
  rowCount={totalRowCount}
  onPaginationChange={(p) => fetchPage(p.pageIndex, p.pageSize)}
  onSortingChange={(s) => fetchSorted(s)}
  onGlobalFilterChange={(q) => fetchSearch(q)}
/>
```

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| data | `TData[]` | ‚Äî | Table data array |
| columns | `ColumnDef<TData>[]` | ‚Äî | Column definitions |
| searchable | `boolean` | `false` | Shows global search input |
| selectable | `boolean` | `false` | Enables row selection checkboxes |
| pageSize | `number` | `10` | Rows per page |
| pageSizeOptions | `number[]` | ‚Äî | Shows a page-size dropdown |
| loading | `boolean` | `false` | Shows skeleton rows while data is fetching |
| emptyText | `string` | `"No results."` | Empty state message |
| rowCount | `number` | ‚Äî | Total row count (required for `manualPagination`) |
| manualPagination | `boolean` | `false` | Delegate pagination to server |
| manualSorting | `boolean` | `false` | Delegate sorting to server |
| manualFiltering | `boolean` | `false` | Delegate global filter to server |
| onPaginationChange | `OnChangeFn<PaginationState>` | ‚Äî | Fires when page/pageSize changes |
| onSortingChange | `OnChangeFn<SortingState>` | ‚Äî | Fires when column sort changes |
| onGlobalFilterChange | `OnChangeFn<string>` | ‚Äî | Fires when the search box changes |
| onRowSelectionChange | `OnChangeFn<RowSelectionState>` | ‚Äî | Fires when row selection changes |

### Accessibility

- Renders a semantic `<table>` with `<thead>` / `<tbody>` / `<th scope="col">` so screen readers announce row/column relationships.
- Sortable column headers use `aria-sort` (`"ascending"`, `"descending"`, `"none"`) and expose a clickable button that meets the 44px touch-target minimum.
- Row-selection checkboxes carry an `aria-label` describing the row ‚Äî the header checkbox announces select-all / deselect-all state.
- The pagination bar exposes `aria-label="Pagination"` with previous/next buttons getting clear labels, and the page-size selector is a standard `<select>` so keyboard and screen-reader interaction follows native semantics.
- The empty state renders inside the tbody region ‚Äî the `emptyText` is announced as soon as the data becomes empty.

```tsx
import { BarChart, LineChart, AreaChart, PieChart, DonutChart, RadarChart } from "@weiui/react/chart";

const data = [
  { name: "Jan", revenue: 4000, expenses: 2400 },
  { name: "Feb", revenue: 3000, expenses: 1398 },
  { name: "Mar", revenue: 2000, expenses: 9800 },
];

<BarChart data={data} dataKeys={["revenue", "expenses"]} title="Revenue vs Expenses" />
<LineChart data={data} dataKeys={["revenue"]} title="Revenue Trend" />
<AreaChart data={data} dataKeys={["revenue"]} title="Revenue Area" />
```

```tsx
const pieData = [
  { name: "Desktop", value: 400 },
  { name: "Mobile", value: 300 },
  { name: "Tablet", value: 100 },
];

<PieChart data={pieData} title="Traffic Sources" />
<DonutChart data={pieData} title="Traffic (Donut)" />
```

### Chart Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| data | `Record<string, unknown>[]` | ‚Äî | Data array. When empty, the chart renders an `EmptyState` instead of the plot. |
| dataKeys | `string[]` | ‚Äî | Keys to chart (Bar/Line/Area/Radar) |
| xKey | `string` | `"name"` | X-axis data key |
| title | `ReactNode` | ‚Äî | Chart title |
| description | `ReactNode` | ‚Äî | Chart description |
| height | `number` | `300` | Chart height in px |
| colors | `string[]` | design tokens | Custom color palette |
| stacked | `boolean \| string` | ‚Äî | BarChart/AreaChart: share a stackId across series. Pass a string to set a custom id. |
| showBrush | `boolean` | ‚Äî | BarChart/LineChart/AreaChart: render a Recharts `<Brush>` for zoom / pan. |
| xAxisFormatter | `(value: unknown) => string` | ‚Äî | Formatter for x-axis ticks (e.g. dates). |
| yAxisFormatter | `(value: unknown) => string` | ‚Äî | Formatter for y-axis ticks + tooltip values (e.g. currency, %). |
| emptyState | `ReactNode` | default `EmptyState` | Replace the empty-data slot. |
| legend | `ReactNode \| null` | `<Legend />` | Replace the default Recharts legend, or pass `null` to hide it. |
| tooltip | `ReactNode \| null` | `<Tooltip />` | Replace the default Recharts tooltip, or pass `null` to hide it. |

### Axis formatters

Use `xAxisFormatter` / `yAxisFormatter` for currency, percentages, compact numbers, or dates. Both the axis ticks and the default tooltip reuse the `yAxisFormatter`.

```tsx
const currency = (v: unknown) => `$${Number(v).toLocaleString()}`;

<BarChart
  data={data}
  dataKeys={["revenue"]}
  yAxisFormatter={currency}
/>
```

### Stacked + brush

`stacked` shares a `stackId` across series. `showBrush` drops in a Recharts `<Brush>` under the plot for zoom / pan ‚Äî the handles are styled with `var(--wui-color-primary)` so they honor the theme.

```tsx
<BarChart data={data} dataKeys={["revenue", "expenses"]} stacked showBrush />
<AreaChart data={data} dataKeys={["a", "b"]} stacked="totals" />
```

### Custom legend / tooltip slots

Pass a Recharts `<Legend>` or `<Tooltip>` element as the `legend` / `tooltip` prop to fully replace the default UI. Pass `null` to remove the slot entirely.

```tsx
import { Legend, Tooltip } from "recharts";

<BarChart
  data={data}
  dataKeys={["revenue"]}
  legend={<Legend verticalAlign="top" iconType="circle" />}
  tooltip={<Tooltip cursor={{ fill: "var(--wui-color-muted)" }} />}
/>

// Hide the default legend
<LineChart data={data} dataKeys={["visits"]} legend={null} />
```

### Empty state

When `data` is an empty array (or undefined), the chart renders a default `EmptyState` in place of the plot. Override the copy with `emptyState`.

```tsx
<LineChart data={[]} dataKeys={["visits"]} />
// or
<LineChart
  data={[]}
  dataKeys={["visits"]}
  emptyState={<EmptyState title="Waiting for data" description="Check back soon." />}
/>
```

### Dark mode

Axis tick labels, grid lines, and polar axes are wired to `var(--wui-color-muted-foreground)` / `var(--wui-color-border)`, so they flip automatically when the theme switches to `data-theme="dark"`. No chart-specific dark CSS is required.

### Accessibility

Every chart automatically mirrors its data into a visually-hidden `<table>` placed alongside the `<svg>`. The table has a caption derived from `title` / `description` and proper `<th>` row/column scopes, so screen-reader users get a readable, linear representation of the chart's numbers.

```html
<!-- Rendered next to the chart SVG -->
<table class="wui-sr-only">
  <caption>Revenue vs Expenses</caption>
  <thead><tr><th>Month</th><th>Revenue</th><th>Expenses</th></tr></thead>
  <tbody><tr><td>Jan</td><td>4000</td><td>2400</td></tr>‚Ä¶</tbody>
</table>
```

Bar / Line / Area chart entrance and tween animations are wrapped in `@media (prefers-reduced-motion: no-preference)`, so users who request reduced motion see the final chart state immediately with no sweeps or easing.

## TreeView

Recursive tree with expand/collapse, keyboard navigation, and selection.

<Preview label="TreeView">
  <TreeViewDemo />
</Preview>

```tsx
import { TreeView, type TreeNode } from "@weiui/react";

const nodes: TreeNode[] = [
  {
    id: "src",
    label: "src",
    children: [
      { id: "src/components", label: "components", children: [
        { id: "src/components/Button.tsx", label: "Button.tsx" },
        { id: "src/components/Input.tsx", label: "Input.tsx" },
      ]},
      { id: "src/index.ts", label: "index.ts" },
    ],
  },
  { id: "package.json", label: "package.json" },
];

<TreeView nodes={nodes} onSelect={(id) => console.log(id)} />
```

### Controlled expanded state

Drive the open/closed state from outside the component by pairing `expanded` with `onExpandedChange`. Useful when expanded state should persist in URL, localStorage, or sync with another view.

```tsx
const [expanded, setExpanded] = useState<string[]>(["src"]);

<TreeView
  nodes={nodes}
  expanded={expanded}
  onExpandedChange={setExpanded}
  onSelect={(id) => console.log(id)}
/>
```

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| nodes | `TreeNode[]` | ‚Äî | Tree data structure |
| selected | `string` | ‚Äî | Currently selected node ID |
| expanded | `string[]` | ‚Äî | Controlled list of expanded node IDs |
| defaultExpanded | `string[]` | `[]` | Initially expanded node IDs (uncontrolled) |
| onExpandedChange | `(ids: string[]) => void` | ‚Äî | Fires when a branch toggles |
| onSelect | `(id: string) => void` | ‚Äî | Selection callback |

### Accessibility

- `role="tree"` with `role="treeitem"` on each node
- `aria-expanded` on branch nodes
- Keyboard: ArrowDown/Up to navigate, ArrowRight to expand, ArrowLeft to collapse, Home/End, Enter/Space to select
- 44px minimum touch target per node

## Pagination

Paginate long lists with ellipsis for large page counts. Works standalone or paired with `<DataTable manualPagination>` for server-driven tables.

<Preview label="Pagination">
  <PaginationDemo />
</Preview>

```tsx
import { Pagination } from "@weiui/react";

<Pagination page={3} totalPages={20} onPageChange={setPage} />
<Pagination page={1} totalPages={100} onPageChange={setPage} siblingCount={2} />
```

### Pagination props

| Prop            | Type                            | Default  | Description                          |
|-----------------|---------------------------------|----------|--------------------------------------|
| `page`          | `number`                        | ‚Äî        | Current 1-indexed page               |
| `totalPages`    | `number`                        | ‚Äî        | Total page count                     |
| `onPageChange`  | `(page: number) => void`        | ‚Äî        | Page change handler                  |
| `siblingCount`  | `number`                        | `1`      | Pages shown around the current one   |

### Pagination accessibility

- Root is `<nav aria-label="Pagination">`.
- Current page button carries `aria-current="page"`.
- Prev / Next expose `aria-label` so screen readers announce the action, not the glyph.
- Ellipses are `aria-hidden="true"` ‚Äî visual cue only.

## Transfer

Dual-list with items moved between a source and target. Use for multi-select when both the pool and the selection must stay visible (permissions, playlist, feature pickers).

<Preview label="Transfer">
  <TransferDemo />
</Preview>

```tsx
import { Transfer } from "@weiui/react";

<Transfer
  sourceItems={[
    { value: "react", label: "React" },
    { value: "vue", label: "Vue" },
    { value: "angular", label: "Angular" },
  ]}
  onChange={(source, target) => console.log({ source, target })}
/>
```

### Controlled target values

Drive the selected side from outside with `targetValues` + `onTargetValuesChange`:

```tsx
const [selected, setSelected] = useState<string[]>(["react"]);

<Transfer
  sourceItems={allFrameworks}
  targetValues={selected}
  onTargetValuesChange={setSelected}
/>
```

### Transfer props

| Prop                    | Type                               | Default       | Description                                            |
|-------------------------|------------------------------------|---------------|--------------------------------------------------------|
| `sourceItems`           | `TransferItem[]`                   | ‚Äî             | Full set of items available                            |
| `targetItems`           | `TransferItem[]`                   | `[]`          | Initial items on the target side (uncontrolled only)   |
| `targetValues`          | `string[]`                         | ‚Äî             | Controlled target values                               |
| `onTargetValuesChange`  | `(values: string[]) => void`       | ‚Äî             | Fires when target changes                              |
| `onChange`              | `(source, target) => void`         | ‚Äî             | Full source + target snapshot                          |
| `sourceTitle`           | `string`                           | `"Available"` | Heading over source list                               |
| `targetTitle`           | `string`                           | `"Selected"`  | Heading over target list                               |

### Transfer accessibility

- Each list is `role="listbox"` + `aria-multiselectable="true"`; rows are `role="option"` with `aria-selected`.
- Lists carry `aria-label="Available items"` / `"Selected items"` so screen readers announce which side they are on.
- Wrapped in `role="group"` with `aria-label="Transfer list"` to form a named landmark.
- Move buttons: `aria-label="Move selected to target"` / `"Move selected to source"`.
