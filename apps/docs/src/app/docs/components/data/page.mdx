# Data Components

DataTable, Charts, and TreeView for data-heavy interfaces.

---

## DataTable

Powered by `@tanstack/react-table` ã sort, filter, paginate, and select rows.

```tsx
import { DataTable, createColumnHelper } from "@weiui/react";

type User = { name: string; email: string; role: string };

const columnHelper = createColumnHelper<User>();
const columns = [
  columnHelper.accessor("name", { header: "Name" }),
  columnHelper.accessor("email", { header: "Email" }),
  columnHelper.accessor("role", { header: "Role" }),
];

<DataTable data={users} columns={columns} searchable pageSize={10} />
```

### Row selection

Pass `selectable` to render a checkbox column plus a header checkbox that selects/deselects the current page. Subscribe to changes with `onRowSelectionChange`.

```tsx
<DataTable
  data={users}
  columns={columns}
  selectable
  onRowSelectionChange={(rows) => console.log("selected:", rows)}
/>
```

### Loading state

When `loading` is set, rows are replaced with skeleton shimmer. Use this while fetching so the table keeps its layout instead of collapsing.

```tsx
<DataTable data={[]} columns={columns} loading />
```

### Page-size selector

`pageSizeOptions` renders a dropdown in the pagination bar so users can pick how many rows to show.

```tsx
<DataTable
  data={users}
  columns={columns}
  pageSize={10}
  pageSizeOptions={[10, 25, 50, 100]}
/>
```

### Server-side data

For datasets that live on the server, opt-out of client-side pagination / sorting / filtering with the `manual*` flags, supply `rowCount`, and respond to state change callbacks by fetching the next page yourself.

```tsx
<DataTable
  data={pageRows}
  columns={columns}
  manualPagination
  manualSorting
  manualFiltering
  rowCount={totalRowCount}
  onPaginationChange={(p) => fetchPage(p.pageIndex, p.pageSize)}
  onSortingChange={(s) => fetchSorted(s)}
  onGlobalFilterChange={(q) => fetchSearch(q)}
/>
```

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| data | `TData[]` | ã | Table data array |
| columns | `ColumnDef<TData>[]` | ã | Column definitions |
| searchable | `boolean` | `false` | Shows global search input |
| selectable | `boolean` | `false` | Enables row selection checkboxes |
| pageSize | `number` | `10` | Rows per page |
| pageSizeOptions | `number[]` | ã | Shows a page-size dropdown |
| loading | `boolean` | `false` | Shows skeleton rows while data is fetching |
| emptyText | `string` | `"No results."` | Empty state message |
| rowCount | `number` | ã | Total row count (required for `manualPagination`) |
| manualPagination | `boolean` | `false` | Delegate pagination to server |
| manualSorting | `boolean` | `false` | Delegate sorting to server |
| manualFiltering | `boolean` | `false` | Delegate global filter to server |
| onPaginationChange | `(state) => void` | ã | Fires when page/pageSize changes |
| onSortingChange | `(state) => void` | ã | Fires when column sort changes |
| onGlobalFilterChange | `(filter: string) => void` | ã | Fires when the search box changes |
| onRowSelectionChange | `(rows: TData[]) => void` | ã | Fires when row selection changes |

## Charts

Six chart types built on Recharts with WeiUI design tokens.

```tsx
import { BarChart, LineChart, AreaChart, PieChart, DonutChart, RadarChart } from "@weiui/react";

const data = [
  { name: "Jan", revenue: 4000, expenses: 2400 },
  { name: "Feb", revenue: 3000, expenses: 1398 },
  { name: "Mar", revenue: 2000, expenses: 9800 },
];

<BarChart data={data} dataKeys={["revenue", "expenses"]} title="Revenue vs Expenses" />
<LineChart data={data} dataKeys={["revenue"]} title="Revenue Trend" />
<AreaChart data={data} dataKeys={["revenue"]} title="Revenue Area" />
```

```tsx
const pieData = [
  { name: "Desktop", value: 400 },
  { name: "Mobile", value: 300 },
  { name: "Tablet", value: 100 },
];

<PieChart data={pieData} title="Traffic Sources" />
<DonutChart data={pieData} title="Traffic (Donut)" />
```

### Chart Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| data | `Record<string, unknown>[]` | ã | Data array |
| dataKeys | `string[]` | ã | Keys to chart (Bar/Line/Area/Radar) |
| xKey | `string` | `"name"` | X-axis data key |
| title | `ReactNode` | ã | Chart title |
| description | `ReactNode` | ã | Chart description |
| height | `number` | `300` | Chart height in px |
| colors | `string[]` | design tokens | Custom color palette |

## TreeView

Recursive tree with expand/collapse, keyboard navigation, and selection.

```tsx
import { TreeView, type TreeNode } from "@weiui/react";

const nodes: TreeNode[] = [
  {
    id: "src",
    label: "src",
    children: [
      { id: "src/components", label: "components", children: [
        { id: "src/components/Button.tsx", label: "Button.tsx" },
        { id: "src/components/Input.tsx", label: "Input.tsx" },
      ]},
      { id: "src/index.ts", label: "index.ts" },
    ],
  },
  { id: "package.json", label: "package.json" },
];

<TreeView nodes={nodes} onSelect={(id) => console.log(id)} />
```

### Controlled expanded state

Drive the open/closed state from outside the component by pairing `expanded` with `onExpandedChange`. Useful when expanded state should persist in URL, localStorage, or sync with another view.

```tsx
const [expanded, setExpanded] = useState<string[]>(["src"]);

<TreeView
  nodes={nodes}
  expanded={expanded}
  onExpandedChange={setExpanded}
  onSelect={(id) => console.log(id)}
/>
```

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| nodes | `TreeNode[]` | ã | Tree data structure |
| selected | `string` | ã | Currently selected node ID |
| expanded | `string[]` | ã | Controlled list of expanded node IDs |
| defaultExpanded | `string[]` | `[]` | Initially expanded node IDs (uncontrolled) |
| onExpandedChange | `(ids: string[]) => void` | ã | Fires when a branch toggles |
| onSelect | `(id: string) => void` | ã | Selection callback |

### Accessibility

- `role="tree"` with `role="treeitem"` on each node
- `aria-expanded` on branch nodes
- Keyboard: ArrowDown/Up to navigate, ArrowRight to expand, ArrowLeft to collapse, Home/End, Enter/Space to select
- 44px minimum touch target per node
