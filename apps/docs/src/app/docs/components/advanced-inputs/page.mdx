# Advanced Inputs

Slider, Rating, InputNumber, and InputOTP — specialized input controls for ranges, ratings, numeric values, and one-time passwords.

---

## Slider

A range slider with min/max/step support and full keyboard navigation.

<Preview label="Slider default">

        <div className="wui-slider__thumb" role="slider" tabIndex={0} aria-valuenow={40} aria-valuemin={0} aria-valuemax={100} aria-label="Slider" style={{ insetInlineStart: "40%" }} />
      </div>
    </div>
  </div>
</Preview>

<Preview label="Slider disabled">
  <div style={{ padding: "0 8px", width: "320px" }}>
    <div className="wui-slider" data-disabled="true">
      <div className="wui-slider__track">
        <div className="wui-slider__fill" style={{ inlineSize: "60%" }} />
        <div className="wui-slider__thumb" role="slider" tabIndex={-1} aria-valuenow={60} aria-valuemin={0} aria-valuemax={100} aria-label="Slider" aria-disabled="true" style={{ insetInlineStart: "60%" }} />

```tsx
import { Slider } from "@weiui/react";

<Slider defaultValue={40} />

```

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| min | `number` | `0` | Minimum value |
| max | `number` | `100` | Maximum value |
| step | `number` | `1` | Step increment |
| value | `number` | — | Controlled value |
| defaultValue | `number` | `0` | Initial uncontrolled value |
| onChange | `(value: number) => void` | — | Called on value change |
| disabled | `boolean` | `false` | Disables the slider |
| label | `string` | `"Slider"` | ARIA label for the thumb |

### Accessibility

- `role="slider"` with `aria-valuenow`, `aria-valuemin`, `aria-valuemax`
- Keyboard: ArrowRight/Up increments, ArrowLeft/Down decrements, Home/End jump to min/max
- 44px minimum touch target height

### Locale & currency formatting

Pass any `Intl.NumberFormatOptions` via the `formatOptions` prop to localise the display. The input still stores a plain number — formatting is applied only on render, and `aria-valuetext` mirrors the formatted string for screen readers.

<Preview label="InputNumber with formatOptions">
  <div className="wui-input-number">
    <button type="button" className="wui-input-number__btn" aria-label="Decrement">−</button>
    <input type="text" className="wui-input-number__input" defaultValue="$1,234.00" aria-label="Amount in USD" role="spinbutton" aria-valuenow={1234} aria-valuetext="$1,234.00" />
    <button type="button" className="wui-input-number__btn" aria-label="Increment">+</button>
  </div>
</Preview>

```tsx
<InputNumber
  defaultValue={1234}
  formatOptions={{ style: "currency", currency: "USD" }}
/>

<InputNumber
  defaultValue={0.75}
  step={0.01}
  formatOptions={{ style: "percent", minimumFractionDigits: 2 }}
/>
```

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| min | `number` | `-Infinity` | Minimum allowed value |
| max | `number` | `Infinity` | Maximum allowed value |
| step | `number` | `1` | Step for increment/decrement |
| value | `number` | — | Controlled value |
| defaultValue | `number` | `0` | Initial uncontrolled value |
| onChange | `(value: number) => void` | — | Called on value change |
| disabled | `boolean` | `false` | Disables the control |
| formatOptions | `Intl.NumberFormatOptions` | — | Currency / percent / locale number formatting |
| locale | `string` | `"en-US"` | BCP-47 locale tag passed to `Intl.NumberFormat` |

### Accessibility

- Input is exposed as `role="spinbutton"` with `aria-valuenow`, `aria-valuemin`, `aria-valuemax`
- `aria-valuetext` announces the formatted value when `formatOptions` is set
- Increment/Decrement buttons have `aria-label`
- ArrowUp/ArrowDown keyboard support on the input
- Buttons auto-disable at min/max bounds
- 44px minimum height touch targets

- Auto-advances focus to next slot on input
- Backspace clears current slot and moves focus back
- Paste distributes characters across slots

## AutoComplete

Text input with a filterable dropdown of suggestions. Anchored to the viewport via Floating UI so it can escape overflow clipping, and supports an async `loading` state.

<Preview label="AutoComplete">
  <AutoCompleteDemo />
</Preview>

```tsx
import { AutoComplete } from "@weiui/react";

const options = [
  { value: "js", label: "JavaScript" },
  { value: "ts", label: "TypeScript" },
  { value: "py", label: "Python" },
];

<AutoComplete options={options} onChange={setValue} />

// Async loading state
<AutoComplete options={results} loading={isFetching} placeholder="Search packages..." />
```

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| options | `{ value: string; label: string }[]` | — | Suggestion list |
| value | `string` | — | Controlled selected option value |
| onChange | `(value: string) => void` | — | Fires when a suggestion is picked |
| onInputChange | `(input: string) => void` | — | Fires as the user types in the text box |
| placeholder | `string` | — | Input placeholder |
| emptyText | `string` | `"No results"` | Message shown when the filter has no matches |
| loading | `boolean` | `false` | Shows a loading row inside the dropdown |
| disabled | `boolean` | `false` | Disables the input |

### Accessibility

- Input exposed as `role="combobox"` with `aria-expanded`, `aria-controls`, `aria-activedescendant`
- Keyboard: ArrowDown opens the list, Arrow keys navigate, Enter selects, Escape closes
- Dropdown anchored with Floating UI — flips and shifts to stay on-screen

## FileUpload

Drag-and-drop or click-to-browse file input with validation hooks.

<Preview label="FileUpload">
  <FileUploadDemo />
</Preview>

```tsx
import { FileUpload } from "@weiui/react";

<FileUpload
  accept="image/*"
  maxSize={5 * 1024 * 1024}
  onChange={(files) => console.log(files)}
  onError={({ reason, file, message }) => {
    toast.error(`${file.name}: ${message}`);
  }}
/>
```

### Error handling

`onError` fires when a drop or selection is rejected — e.g. wrong `accept` MIME type (`reason: "type"`), exceeded `maxSize` (`"size"`), or exceeded `maxFiles` (`"count"`). The handler receives `{ reason, file, message }` so you can pick whether to toast, inline-render, or silently log. An inline error is also rendered inside the dropzone automatically.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| accept | `string` | — | MIME types / extensions (same as `<input type="file">`) |
| multiple | `boolean` | `false` | Allow multiple files |
| maxSize | `number` | — | Max bytes per file |
| maxFiles | `number` | — | Max file count when `multiple` is true |
| onChange | `(files: File[]) => void` | — | Called when files are added or removed |
| onError | `(err: { reason: "type" \| "size" \| "count"; file: File; message: string }) => void` | — | Called when a file is rejected |
| label | `string` | `"Click or drag files here"` | Dropzone label text |
| hint | `string` | — | Optional helper text under the label |
| disabled | `boolean` | `false` | Disables drop + browse |

### Accessibility

- Dropzone acts as a button: `role="button"`, activates the hidden file input on Enter/Space
- Hint text renders inside the dropzone so screen readers pick it up via the `aria-label`
- Inline error region uses `role="alert"` for immediate announcement
