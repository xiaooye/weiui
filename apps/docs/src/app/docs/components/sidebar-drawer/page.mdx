# Sidebar & Drawer

Navigation panels and slide-in overlays for app layouts.

---

## Sidebar

A collapsible navigation panel that sticks to the viewport edge.

<Preview label="Sidebar">
  <SidebarDemo />
</Preview>

```tsx
import { Sidebar, SidebarHeader, SidebarContent, SidebarFooter, SidebarItem } from "@weiui/react";

<Sidebar aria-label="Main navigation">
  <SidebarHeader>App Name</SidebarHeader>
  <SidebarContent>
    <SidebarItem active>Dashboard</SidebarItem>
    <SidebarItem>Settings</SidebarItem>
  </SidebarContent>
  <SidebarFooter>v1.0.0</SidebarFooter>
</Sidebar>
```

### Sidebar Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| open | `boolean` | — | Controls whether the sidebar is collapsed to icons only (`true`) or expanded with full labels (`false`). |
| defaultOpen | `boolean` | `false` | Initial collapsed state (uncontrolled). Default is expanded. |
| onOpenChange | `(open: boolean) => void` | — | Called when the collapsed state toggles. |

### Collapsed (icon-only) mode

When the sidebar is collapsed, labels are visually hidden and the sidebar narrows to fit icons only — the layout transition is CSS-driven so the adjacent content reflows without layout shift. Screen readers still announce the full label via `aria-label`.

```tsx
<Sidebar defaultOpen>
  <SidebarContent>
    <SidebarItem icon={<HomeIcon />}>Home</SidebarItem>
    <SidebarItem icon={<SettingsIcon />}>Settings</SidebarItem>
  </SidebarContent>
</Sidebar>
```

Toggle by flipping `open` / calling `onOpenChange`. In CSS, the mode is a `data-collapsed` attribute — you can target it (e.g. `.wui-sidebar[data-collapsed] .brand-text { display: none }`) to tailor the visual further.

### SidebarItem Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| active | `boolean` | `false` | Highlights as current page; sets `aria-current="page"`. |
| icon | `ReactNode` | — | Icon rendered before the label; remains visible in collapsed mode. |

### Accessibility

- Root renders a semantic `<aside>` — always pass an `aria-label` (e.g. `"Main navigation"`) so screen readers can distinguish it from other asides on the page.
- Active items set `aria-current="page"` so assistive tech announces the current location.
- Labels stay in the DOM during collapsed mode (visually hidden via `wui-sr-only`), so screen readers keep announcing each item's name even when only icons are visible.
- Arrow-key navigation works on items that opt into list semantics; standard Tab order applies for the default button-list pattern.

## Drawer

A slide-in panel from any edge with overlay and focus trap.

<Preview label="Drawer">
  <DrawerDemo />
</Preview>

```tsx
import { Drawer, DrawerTrigger, DrawerContent, DrawerHeader, DrawerClose } from "@weiui/react";

<Drawer side="right">
  <DrawerTrigger>Open Drawer</DrawerTrigger>
  <DrawerContent>
    <DrawerHeader>Settings</DrawerHeader>
    <p>Your drawer content.</p>
    <DrawerClose>Close</DrawerClose>
  </DrawerContent>
</Drawer>
```

### Props

`side` and open-state props live on the `<Drawer>` root.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| side | `"left" \| "right" \| "top" \| "bottom"` | `"right"` | Slide direction |
| open | `boolean` | — | Controlled open state |
| defaultOpen | `boolean` | `false` | Initial uncontrolled state |
| onOpenChange | `(open: boolean) => void` | — | Open/close callback |

### Accessibility

- `role="dialog"` with `aria-modal="true"` on content
- Focus trap while open, Escape to close
- Focus returns to trigger on close
- Slide animation respects `prefers-reduced-motion`
- Rendered via Portal, so the drawer is never clipped by a scrolling parent and sits above all app chrome
