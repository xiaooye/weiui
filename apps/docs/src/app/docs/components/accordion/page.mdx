# Accordion

Collapsible content sections — single or multiple expand mode.

---

```tsx
import { Accordion, AccordionItem, AccordionTrigger, AccordionContent } from "@weiui/react";

<Accordion type="single">
  <AccordionItem value="faq-1">
    <AccordionTrigger value="faq-1">What is WeiUI?</AccordionTrigger>
    <AccordionContent value="faq-1">
      A business-level design system with CSS, headless, and styled React layers.
    </AccordionContent>
  </AccordionItem>
  <AccordionItem value="faq-2">
    <AccordionTrigger value="faq-2">Is it accessible?</AccordionTrigger>
    <AccordionContent value="faq-2">
      Yes — WCAG AAA compliant with full keyboard support.
    </AccordionContent>
  </AccordionItem>
</Accordion>

// Multiple expand
<Accordion type="multiple" defaultExpanded={["item-1"]}>
  ...
</Accordion>
```

### Controlled expanded state

Both `single` and `multiple` modes accept a controlled `value` plus `onValueChange`. The `value` is `string | null` in single mode and `string[]` in multiple mode.

```tsx
// single — controlled
const [open, setOpen] = useState<string | null>("faq-1");

<Accordion type="single" value={open} onValueChange={setOpen}>
  {/* ... */}
</Accordion>

// multiple — controlled
const [open, setOpen] = useState<string[]>(["faq-1"]);

<Accordion type="multiple" value={open} onValueChange={setOpen}>
  {/* ... */}
</Accordion>
```

### Keyboard

- ArrowDown / ArrowUp move focus between triggers (wraps at boundaries)
- Home / End jump to the first / last trigger
- Enter / Space toggles the focused item
- Disabled items are skipped during navigation

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| type | `"single" \| "multiple"` | `"single"` | Allow one or many open at once |
| value | `string \| string[] \| null` | — | Controlled expanded value(s) |
| onValueChange | `(value: string \| string[] \| null) => void` | — | Called when expanded changes |
| defaultExpanded | `string[]` | `[]` | Initially expanded items (uncontrolled) |

### Accessibility

- Trigger uses `aria-expanded` and `aria-controls`
- Content uses `role="region"`
- Arrow-key roving between triggers, Home/End jumps, wraps at edges
- Chevron icon rotates on expand (respects `prefers-reduced-motion`)
