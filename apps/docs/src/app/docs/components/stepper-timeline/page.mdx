# Stepper & Timeline

Step progress indicators and chronological event displays.

---

## Stepper

Multi-step progress indicator showing active, completed, and pending steps.

<Preview label="Stepper horizontal">
  <StepperDemo />
</Preview>

```tsx
import { Stepper, Step, StepSeparator } from "@weiui/react";

// Explicit indices
<Stepper activeStep={1}>
  <Step index={0} label="Account" description="Create your account" />
  <StepSeparator index={0} />
  <Step index={1} label="Profile" description="Fill in details" />
  <StepSeparator index={1} />
  <Step index={2} label="Review" description="Confirm setup" />
</Stepper>

// Auto-inferred indices — Stepper walks its children and assigns an
// index to each <Step>; separators take the preceding step's index.
<Stepper activeStep={1}>
  <Step label="Account" description="Create your account" />
  <StepSeparator />
  <Step label="Profile" description="Fill in details" />
  <StepSeparator />
  <Step label="Review" description="Confirm setup" />
</Stepper>
```

You can mix the two: an explicit `index` on any child overrides the auto-assigned value.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| activeStep | `number` | — | Current active step index |
| orientation | `"horizontal" \| "vertical"` | `"horizontal"` | Layout direction |

### Step / StepSeparator Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| index | `number` | auto | Explicit step index. Omit to let `Stepper` auto-infer from child order. |
| label | `ReactNode` | — | Step title (Step only) |
| description | `ReactNode` | — | Step supporting text (Step only) |

### Accessibility

- The active step gets `aria-current="step"`, matching the ARIA Authoring Practices pattern, so screen readers announce which step the user is on.
- Completed / active state is conveyed via `data-completed` / `data-active` for CSS targeting while keeping the decorative checkmark (`\u2713`) out of announcements.
- Connecting separator lines are `aria-hidden="true"` — they are decorative, not content.
- Pass `aria-label` on the `<Stepper>` wrapper if the step flow needs a name (e.g. `"Signup progress"`).

## Timeline

Chronological event display with dots and connecting lines.

<Preview label="Timeline">
  <TimelineDemo />
</Preview>

```tsx
import { Timeline, TimelineItem } from "@weiui/react";

<Timeline>
  <TimelineItem title="Application submitted" description="Your application was received" time="Jan 15" />
  <TimelineItem title="Under review" description="Being reviewed by our team" time="Jan 20" />
</Timeline>
```

### TimelineItem Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| title | `ReactNode` | — | Event heading |
| description | `ReactNode` | — | Supporting event text |
| time | `ReactNode` | — | Timestamp text (e.g. `"Jan 15"` or a `<time>` element) |

### Accessibility

- Uses semantic `<ol>` and `<li>` elements
- Connecting lines are `aria-hidden="true"`
- Server component — no `"use client"` required
