## Pagination

Page navigation with ellipsis for large page counts.

```tsx
import { Pagination } from "@weiui/react";

<Pagination page={3} totalPages={20} onPageChange={setPage} />
<Pagination page={1} totalPages={100} onPageChange={setPage} siblingCount={2} />
```

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| page | `number` | — | Current page |
| totalPages | `number` | — | Total page count |
| onPageChange | `(page: number) => void` | — | Page change handler |
| siblingCount | `number` | `1` | Pages shown around current |

## Transfer

Dual-list with items moved between source and target.

```tsx
import { Transfer } from "@weiui/react";

<Transfer
  sourceItems={[
    { value: "react", label: "React" },
    { value: "vue", label: "Vue" },
    { value: "angular", label: "Angular" },
  ]}
  onChange={(source, target) => console.log({ source, target })}
/>
```

### Controlled target values

Drive the selected-side from the outside with `targetValues` + `onTargetValuesChange`. Useful when the selection is part of form state or must be persisted.

```tsx
const [selected, setSelected] = useState<string[]>(["react"]);

<Transfer
  sourceItems={allFrameworks}
  targetValues={selected}
  onTargetValuesChange={setSelected}
/>
```

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| sourceItems | `{ value: string; label: string }[]` | — | Full set of items available |
| targetValues | `string[]` | — | Controlled selected values on the target side |
| defaultTargetValues | `string[]` | `[]` | Initial uncontrolled selected values |
| onTargetValuesChange | `(values: string[]) => void` | — | Fires when the target list changes |

## Splitter

Resizable split panes — horizontal or vertical.

```tsx
import { Splitter } from "@weiui/react";

<Splitter defaultSizes={[60, 40]} style={{ height: "400px" }}>
  <div>Left panel</div>
  <div>Right panel</div>
</Splitter>

<Splitter orientation="vertical" minSize={20}>
  <div>Top</div>
  <div>Bottom</div>
</Splitter>
```

### Controlled sizes

Both panes can be driven from outside via `sizes` + `onSizesChange`. Sizes are an array of percentages that must sum to 100.

```tsx
const [sizes, setSizes] = useState<[number, number]>([30, 70]);

<Splitter sizes={sizes} onSizesChange={setSizes}>
  <Nav />
  <Editor />
</Splitter>
```

### Accessibility
- Handle uses `role="separator"` with `aria-orientation` and `aria-valuenow`
- Keyboard: ArrowLeft/Right (horizontal) or ArrowUp/Down (vertical) to resize

## BottomNav

Mobile bottom navigation bar — fixed to viewport bottom. Adds `env(safe-area-inset-bottom)` padding automatically so the bar clears the iOS home indicator.

```tsx
import { BottomNav, BottomNavItem } from "@weiui/react";

<BottomNav>
  <BottomNavItem active icon="🏠" label="Home" onClick={() => navigate("/")} />
  <BottomNavItem icon="🔍" label="Search" onClick={() => navigate("/search")} />
  <BottomNavItem icon="👤" label="Profile" onClick={() => navigate("/profile")} />
</BottomNav>
```

The safe-area inset is applied via CSS on `.wui-bottom-nav` and is therefore zero on platforms without a bottom inset. No consumer-side configuration is needed.

## SpeedDial

Floating action button that expands into multiple actions.

```tsx
import { SpeedDial } from "@weiui/react";

<SpeedDial
  actions={[
    { id: "add", icon: "+", label: "Add item", onClick: () => add() },
    { id: "edit", icon: "✏", label: "Edit", onClick: () => edit() },
    { id: "share", icon: "↗", label: "Share", onClick: () => share() },
  ]}
/>
```

### Keyboard
- Enter / Space on the FAB opens the action list and moves focus to the first action
- ArrowUp / ArrowDown (or ArrowLeft / ArrowRight depending on `direction`) roves focus between actions
- Home / End jumps to the first / last action
- Escape closes the menu and returns focus to the FAB
- Focus is locked inside the open speed dial — Tab stays within the actions

## ToggleGroup

Grouped toggle buttons — single or multiple selection with full roving-tabindex keyboard support.

```tsx
import { ToggleGroup, ToggleGroupItem } from "@weiui/react";

// Single — one item active at a time
<ToggleGroup type="single" defaultValue="left">
  <ToggleGroupItem value="left">Left</ToggleGroupItem>
  <ToggleGroupItem value="center">Center</ToggleGroupItem>
  <ToggleGroupItem value="right">Right</ToggleGroupItem>
</ToggleGroup>

// Multiple — any combination active
<ToggleGroup type="multiple" defaultValue={["bold"]}>
  <ToggleGroupItem value="bold">B</ToggleGroupItem>
  <ToggleGroupItem value="italic">I</ToggleGroupItem>
  <ToggleGroupItem value="underline">U</ToggleGroupItem>
</ToggleGroup>
```

### Keyboard
- ArrowLeft / ArrowRight (and ArrowUp / ArrowDown) rove focus between items, wrapping at the ends
- Home / End jump to the first / last item
- Enter / Space toggles the focused item
- Only one item in the group is tabbable at a time (roving tabindex)

## ButtonGroup

```tsx

<ButtonGroup>
  <Button variant="outline">Copy</Button>
  <Button variant="outline">Cut</Button>
  <Button variant="outline">Paste</Button>
</ButtonGroup>
```
