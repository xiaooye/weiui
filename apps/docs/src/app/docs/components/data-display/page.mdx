# Data Display

Static surfaces for rendering information — containers, labels, identity, and loading placeholders. All four primitives are Server Components with zero client JavaScript unless an interactive child opts in.

## Card

A container with optional header, content, and footer slots. Cards use `container-type: inline-size` so child layouts can respond to the card's width, not the viewport.

<Preview label="Card with header, content, and footer">
  <CardDemo />
</Preview>

```tsx
import { Card, CardHeader, CardContent, CardFooter, Button } from "@weiui/react";

<Card>
  <CardHeader>
    <strong>Project update</strong>
  </CardHeader>
  <CardContent>
    Your build finished in 3.2s. All 128 tests passed.
  </CardContent>
  <CardFooter>
    <Button size="sm">Deploy</Button>
    <Button variant="outline" size="sm">Dismiss</Button>
  </CardFooter>
</Card>
```

### Variants

| variant     | Use for                                             |
|-------------|-----------------------------------------------------|
| `elevated`  | Default — subtle shadow, raised feel                |
| `outlined`  | Border only, no shadow — dense layouts              |
| `filled`    | Sunken surface — summary panels inside other chrome |

### Interactive cards

Add `asChild` and wrap a link to make the entire card clickable while preserving semantics:

```tsx
<Card variant="outlined" asChild>
  <a href="/settings">
    <CardHeader>Account</CardHeader>
    <CardContent>Profile, password, and sessions</CardContent>
  </a>
</Card>
```

### Card props

| Prop       | Type                                        | Default     | Description                           |
|------------|---------------------------------------------|-------------|---------------------------------------|
| `variant`  | `"elevated" \| "outlined" \| "filled"`      | `"elevated"`| Visual style                          |
| `asChild`  | `boolean`                                   | `false`     | Render as the child element (Slot)    |
| `padding`  | `"none" \| "sm" \| "md" \| "lg"`            | `"md"`      | Internal padding applied to children  |

### Accessibility

- Cards are styling primitives — they should not override semantic roles. A `<Card>` is a `<div>` by default.
- When `asChild` wraps a link or button, focus ring and hover elevation follow the interactive child.
- Do not nest landmarks inside Cards — prefer `<section>` / `<article>` wrappers when applicable.

## Badge

A compact label for status, counts, or categorisation. Supports three variants across seven colors with automatic AAA contrast for all combinations.

<Preview label="Badge variants and colors">
  <BadgeDemo />
</Preview>

```tsx
import { Badge } from "@weiui/react";

<Badge variant="solid">Solid</Badge>
<Badge variant="soft" color="success">Active</Badge>
<Badge variant="outline" color="warning">Pending</Badge>
```

### Dot indicator

```tsx
<Badge color="success" dot>
  <span className="wui-badge__dot" aria-hidden /> Online
</Badge>
```

### With icon

```tsx
import { CheckCircle } from "@weiui/icons";

<Badge variant="soft" color="success">
  <CheckCircle size={12} aria-hidden /> Verified
</Badge>
```

### Badge props

| Prop      | Type                                                          | Default   | Description            |
|-----------|---------------------------------------------------------------|-----------|------------------------|
| `variant` | `"solid" \| "soft" \| "outline"`                              | `"solid"` | Fill style             |
| `color`   | `"primary" \| "secondary" \| "success" \| "warning" \| "destructive" \| "info" \| "neutral"` | `"neutral"` | Semantic color tone |
| `size`    | `"sm" \| "md"`                                                | `"md"`    | Label size             |

### Accessibility

- Badges are decorative spans. For status information, pair with visible text or an `aria-label`.
- When counting notifications, prefer `<span className="wui-badge"><VisuallyHidden>3 unread messages</VisuallyHidden>3</span>` so screen readers announce intent, not just the number.
- Color alone is never the sole differentiator — the label text carries meaning.

## Avatar

User or entity identity. Falls back through a chain: `src` → `AvatarFallback` → initials from `name` → name-derived color background.

<Preview label="Avatar sizes + image fallback">
  <AvatarDemo />
</Preview>

```tsx
import { Avatar, AvatarImage, AvatarFallback } from "@weiui/react";

<Avatar size="lg">
  <AvatarImage src="/u/jane.jpg" alt="Jane Doe" />
  <AvatarFallback>JD</AvatarFallback>
</Avatar>

{/* With auto-initials and deterministic color */}
<Avatar name="Jane Doe" />
```

### AvatarGroup

Stack overlapping avatars with a `+N` overflow chip for the rest.

<Preview label="AvatarGroup with overflow">
  <AvatarGroupDemo />
</Preview>

```tsx
import { Avatar, AvatarGroup } from "@weiui/react";

<AvatarGroup max={4}>
  <Avatar name="Ana Lo" />
  <Avatar name="Ben Yu" />
  <Avatar name="Cai Park" />
  <Avatar name="Diana Li" />
  <Avatar name="Emil Gerova" />
  <Avatar name="Fan Sung" />
</AvatarGroup>
```

### Avatar props

| Prop            | Type                              | Default | Description                                            |
|-----------------|-----------------------------------|---------|--------------------------------------------------------|
| `size`          | `"sm" \| "md" \| "lg" \| "xl"`    | `"md"`  | Avatar size                                            |
| `name`          | `string`                          | —       | Auto-initials + deterministic background color        |
| `src`           | `string`                          | —       | Image URL; on error, falls back gracefully             |
| `colorFromName` | `boolean`                         | auto    | Toggle the hash-based color (default on when `name`)   |

### Accessibility

- Always supply `alt` on `<AvatarImage>` or set `name` — otherwise wrap initials in `<VisuallyHidden>`.
- When an avatar represents a live status, pair with a separate status dot and announce the status, not the image.
- AvatarGroup announces its overflow label (`+3`) to screen readers; provide an `aria-label` like `"5 project collaborators"` on the group itself for context.

## Skeleton

Loading placeholder that mirrors the final layout's rectangles, text lines, and circles. Animates subtly via background-position; animation is gated on `prefers-reduced-motion`.

<Preview label="Skeleton loading states">
  <SkeletonDemo />
</Preview>

### Card skeleton

Mirror a real `<Card>` while its data is in-flight:

```tsx
<Card>
  <CardHeader><Skeleton variant="text" style={{ width: "40%" }} /></CardHeader>
  <CardContent>
    <Skeleton style={{ height: 120, width: "100%" }} />
    <Skeleton variant="text" style={{ width: "90%" }} />
    <Skeleton variant="text" style={{ width: "60%" }} />
  </CardContent>
</Card>
```

### Avatar + text skeleton

```tsx
<div style={{ display: "flex", gap: 12, alignItems: "center" }}>
  <Skeleton variant="circle" style={{ width: 48, height: 48 }} />
  <div style={{ flex: 1 }}>
    <Skeleton variant="text" style={{ width: "70%" }} />
    <Skeleton variant="text" style={{ width: "40%" }} />
  </div>
</div>
```

### Table skeleton

```tsx
{Array.from({ length: 5 }).map((_, i) => (
  <div key={i} style={{ display: "grid", gridTemplateColumns: "1fr 2fr 1fr", gap: 12 }}>
    <Skeleton variant="text" />
    <Skeleton variant="text" />
    <Skeleton variant="text" style={{ width: "50%" }} />
  </div>
))}
```

### Skeleton props

| Prop      | Type                              | Default   | Description                                       |
|-----------|-----------------------------------|-----------|---------------------------------------------------|
| `variant` | `"rect" \| "text" \| "circle"`    | `"rect"`  | Shape                                             |
| `width`   | `string \| number`                | —         | Shortcut for explicit width (else uses container) |
| `height`  | `string \| number`                | —         | Shortcut for explicit height                      |
| `count`   | `number`                          | `1`       | Render N clones (useful for paragraph skeletons)  |

### Accessibility

- `aria-hidden="true"` is automatically set — skeletons are purely decorative.
- Wrap the skeleton region in a parent `aria-busy="true"` and swap to real content when loaded; screen readers track the change.
- The shimmer animation is disabled when `prefers-reduced-motion: reduce` — the skeleton remains visible as a static rectangle.

## When to use which

| Need                                 | Use                              |
|--------------------------------------|----------------------------------|
| Group related content on a surface   | **Card**                         |
| Show a count, status, or category    | **Badge**                        |
| Identify a person or entity          | **Avatar** (or **AvatarGroup**)  |
| Hint that content is loading         | **Skeleton**                     |
| A transient foreground alert         | See [Feedback](/docs/components/feedback) → Alert |
| A dismissible pill with actions      | See [Chip](/docs/components/toast-chip-progress#chip) |

Avoid using a Card as a page section — semantic `<section>` with a heading is cheaper and more accessible. Reach for Card when you need the visual affordance of "grouped, separable unit" (e.g. a list of projects, feed items).
