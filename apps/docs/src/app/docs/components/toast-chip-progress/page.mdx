# Toast, Chip & ProgressBar

## Toast

Toast is a notification system driven by an external store. The docs site already has a `<Toaster />` mounted in the root layout, so the buttons below call the real `toast()` API.

<Preview
  label="Toast variants"
  code={`import { toast, Button } from "@weiui/react";

<Button onClick={() => toast.success("Saved!")}>Success</Button>
<Button onClick={() => toast.error("Something failed")}>Error</Button>
<Button onClick={() => toast.warning("Heads up")}>Warning</Button>

  <ToastDemo />

Place `<Toaster />` once near the root of your app (e.g. in a layout). Then call `toast()` from anywhere.

```tsx
// In your root layout
import { Toaster } from "@weiui/react";

export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        {children}
        <Toaster />
      </body>
    </html>
  );
}
```

```tsx
// Anywhere in your app
import { toast } from "@weiui/react";

// Default
toast("File saved successfully");

// With description
toast("Update available", { description: "Refresh to apply the latest changes." });

// Variants
toast.success("Payment received");
toast.error("Something went wrong", { description: "Please try again later." });
toast.warning("Session expiring soon");

// Custom duration (ms). Pass 0 to disable auto-dismiss.
toast("Manual dismiss only", { duration: 0 });

// With an inline action button
toast("Message archived", {
  action: {
    label: "Undo",
    onClick: () => restoreMessage(),
  },
});
```

### Position

Configure where the stack appears on `<Toaster>`. All six corners are supported.

```tsx
<Toaster position="top-right" />
<Toaster position="bottom-center" />
```

### Props — `toast()`

| Argument | Type | Default | Description |
|----------|------|---------|-------------|
| title | `string` | — | Required notification text |
| opts.description | `string` | — | Optional supporting text |
| opts.variant | `"default" \| "success" \| "destructive" \| "warning"` | `"default"` | Visual style |
| opts.duration | `number` | `5000` | Auto-dismiss delay in ms. `0` disables auto-dismiss. |
| opts.action | `{ label: string; onClick: () => void }` | — | Renders an inline action button on the toast |

### Props — `<Toaster>`

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| position | `"top-left" \| "top-center" \| "top-right" \| "bottom-left" \| "bottom-center" \| "bottom-right"` | `"bottom-right"` | Stack anchor corner |

### Accessibility

- `<Toaster>` has `role="region"` and `aria-label="Notifications"`
- Each toast has `role="alert"` so screen readers announce it immediately
- Close button has `aria-label="Close notification"`
- Action button (when provided) is keyboard-focusable and activated by Enter/Space

## Chip

Like Badge but interactive — can be dismissed with a remove button.

<Preview label="Chip with onRemove">
  <ChipDemo />
</Preview>

```tsx
import { Chip } from "@weiui/react";

// Static
<Chip>Default</Chip>
<Chip color="primary">Primary</Chip>
<Chip color="success">Success</Chip>
<Chip color="destructive">Error</Chip>

// Dismissible
<Chip onRemove={() => handleRemove("typescript")}>TypeScript</Chip>
```

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| children | `ReactNode` | — | Chip label content |
| color | `"default" \| "primary" \| "success" \| "destructive"` | `"default"` | Color variant |
| onRemove | `() => void` | — | When provided, renders a remove button |

```tsx
import { ProgressBar } from "@weiui/react";

<ProgressBar value={60} />
<ProgressBar value={80} color="success" />
<ProgressBar value={40} color="warning" size="sm" label="Upload progress" />
<ProgressBar value={25} color="destructive" size="lg" />

// Indeterminate (no known value)
<ProgressBar indeterminate label="Loading data" />
```

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| value | `number` | `0` | Current progress value |
| max | `number` | `100` | Maximum value |
| size | `"sm" \| "md" \| "lg"` | `"md"` | Track height |
| color | `"primary" \| "success" \| "destructive" \| "warning"` | `"primary"` | Bar color |
| indeterminate | `boolean` | — | Show animated indeterminate state |
| label | `string` | `"Progress"` | Accessible label for screen readers |

### Accessibility

- `role="progressbar"` with `aria-valuenow`, `aria-valuemin`, `aria-valuemax`
- `aria-valuenow` is omitted in indeterminate mode (per ARIA spec)
- `aria-label` set via the `label` prop
