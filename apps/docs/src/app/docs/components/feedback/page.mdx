# Feedback Components

Components for communicating status, loading states, and empty content to users.

## Alert

<Preview label="Alert variants">
  <AlertDemo />
</Preview>

```tsx
import { Alert, AlertTitle, AlertDescription } from "@weiui/react";

<Alert variant="info">
  <AlertTitle>Info</AlertTitle>
  <AlertDescription>This is an informational message.</AlertDescription>
</Alert>

<Alert variant="success">
  <AlertTitle>Success</AlertTitle>
  <AlertDescription>Your changes have been saved.</AlertDescription>
</Alert>

<Alert variant="warning">
  <AlertTitle>Warning</AlertTitle>
  <AlertDescription>This action cannot be undone.</AlertDescription>
</Alert>

<Alert variant="destructive">
  <AlertTitle>Error</AlertTitle>
  <AlertDescription>Something went wrong. Please try again.</AlertDescription>
</Alert>
```

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| variant | `"info" \| "success" \| "warning" \| "destructive"` | `"info"` | Visual style |
| children | `ReactNode` | — | Alert content |

### Accessibility

- Alert uses `role="alert"` on the root so screen readers announce the message when it appears.
- `<AlertTitle>` renders as a heading (`<h5>` by default) so assistive tech users can navigate to it via heading shortcuts.
- Destructive variant is paired with an icon inside the content slot — never convey the severity by color alone.
- Contrast for every variant is validated against the 7:1 AAA threshold at build time.

## Spinner

<Preview label="Spinner sizes">
  <SpinnerDemo />
</Preview>

```tsx
import { Spinner } from "@weiui/react";

<Spinner size="sm" />
<Spinner size="md" />
<Spinner size="lg" />
<Spinner label="Saving changes" />
```

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| size | `"sm" \| "md" \| "lg"` | `"md"` | Spinner size |
| label | `string` | `"Loading"` | Accessible label (screen readers) |

### Accessibility

- `role="status"` announces the loading state to screen readers
- `aria-label` set to the `label` prop
- Visible label text hidden via `wui-sr-only`
- Rotation animation is wrapped in `@media (prefers-reduced-motion: no-preference)` — users who prefer reduced motion see a static circle instead of a spin, while the `role="status"` announcement still plays

## EmptyState

<Preview label="Empty state">
  <EmptyStateDemo />
</Preview>

```tsx
import { EmptyState } from "@weiui/react";

<EmptyState
  title="No results found"
  description="Try adjusting your search or filters."
  action={<Button variant="outline">Clear filters</Button>}
/>
```

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| title | `string` | — | Required heading text |
| description | `string` | — | Optional supporting text |
| icon | `ReactNode` | — | Optional icon above the title |
| action | `ReactNode` | — | Optional action element |

### Accessibility

- `title` renders inside an `<h3>` so keyboard users can jump to it with heading shortcuts.
- Mark the decorative `icon` slot with `aria-hidden="true"` when you pass one — the `title` carries the meaning, and letting screen readers describe the glyph adds noise.
- The action slot accepts any focusable element; pair it with a clear call to action (e.g. "Clear filters", "Add your first project") so screen reader users know how to recover from the empty state.
