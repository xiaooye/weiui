# Feedback

Components for communicating status, loading, and empty content. Each primitive is small, accessible by default, and composes cleanly with Cards, Dialogs, and layout containers.

## Alert

Inline messaging that is not time-sensitive — confirm, warn, or explain. Every variant ships a default icon (customisable or set to `null`), and all four variants pass WCAG AAA contrast.

<Preview label="Alert variants">
  <AlertDemo />
</Preview>

```tsx
import { Alert, AlertTitle, AlertDescription } from "@weiui/react";

<Alert variant="info">
  <AlertTitle>Info</AlertTitle>
  <AlertDescription>We just updated our terms.</AlertDescription>
</Alert>
```

### With action + dismiss

Provide a recovery path for the message (e.g. "Undo") and let users dismiss when they're done reading.

<Preview label="Alert with action + dismiss">
  <AlertDismissibleDemo />
</Preview>

```tsx
<Alert
  variant="info"
  dismissible
  onDismiss={() => setShown(false)}
  action={<Button size="sm" variant="outline">Undo</Button>}
>
  <AlertTitle>Draft saved</AlertTitle>
  <AlertDescription>Autosaved 12 seconds ago.</AlertDescription>
</Alert>
```

### Hide or replace the icon

```tsx
{/* Remove the default icon */}
<Alert variant="warning" icon={null}>
  <AlertTitle>Heads up</AlertTitle>
  <AlertDescription>No icon in this one.</AlertDescription>
</Alert>

{/* Swap in your own */}
import { Zap } from "@weiui/icons";

<Alert variant="info" icon={<Zap size={18} aria-hidden />}>
  <AlertTitle>New feature</AlertTitle>
  <AlertDescription>Workflows ship today.</AlertDescription>
</Alert>
```

### Alert props

| Prop          | Type                                                       | Default   | Description                                          |
|---------------|------------------------------------------------------------|-----------|------------------------------------------------------|
| `variant`     | `"info" \| "success" \| "warning" \| "destructive"`        | `"info"`  | Tone + default icon                                  |
| `icon`        | `ReactNode \| null`                                        | default   | Replace icon or set `null` to hide                   |
| `dismissible` | `boolean`                                                  | `false`   | Render a close "×" button                            |
| `onDismiss`   | `() => void`                                               | —         | Fired when the close button is clicked               |
| `action`      | `ReactNode`                                                | —         | Trailing action slot (usually a Button)              |

### Alert accessibility

- Root has `role="alert"` so screen readers announce when the Alert mounts. If the Alert is present on load (not inserted dynamically), consider using a plain Card instead.
- `<AlertTitle>` is an `<h5>` — keep it concise and leading-focused.
- The destructive variant carries an icon *and* a title so meaning is never conveyed by color alone.
- The close button uses `aria-label="Dismiss"` and keyboard-dismisses on Enter/Space.

## Spinner

An indeterminate loading indicator. Reach for it when a short action is in-flight (loading a table, saving a form). For long or multi-step operations, prefer `<ProgressBar>`.

<Preview label="Spinner sizes">
  <SpinnerDemo />
</Preview>

```tsx
import { Spinner } from "@weiui/react";

<Spinner size="sm" />
<Spinner size="md" label="Saving changes" />
<Spinner size="lg" />
```

### Inline inside a button

```tsx
<Button disabled>
  <Spinner size="sm" label="Submitting" /> Submitting…
</Button>
```

### Overlay inside a loading region

```tsx
<div style={{ position: "relative", minHeight: 240 }} aria-busy={loading}>
  {loading && (
    <div style={{ position: "absolute", inset: 0, display: "grid", placeItems: "center" }}>
      <Spinner size="lg" label="Loading projects" />
    </div>
  )}
  <ProjectList data={data} />
</div>
```

### Spinner props

| Prop     | Type                              | Default      | Description                          |
|----------|-----------------------------------|--------------|--------------------------------------|
| `size`   | `"sm" \| "md" \| "lg"`            | `"md"`       | Diameter                             |
| `label`  | `string`                          | `"Loading"`  | Screen-reader announcement           |

### Spinner accessibility

- Renders with `role="status"` + `aria-label={label}`. Polite assistive-tech announcement, not disruptive.
- The visible label is kept in the DOM (inside `wui-sr-only`) so it still reaches screen readers; it is not rendered visually.
- Spin animation is wrapped in `@media (prefers-reduced-motion: no-preference)` — users who prefer reduced motion see a static circle but still hear the status.

## EmptyState

Communicates absence of data with purpose — tell the user what's missing and what to do. Every empty state needs a title; description, illustration, and action are optional but recommended.

<Preview label="Empty state with illustration + action">
  <EmptyStateDemo />
</Preview>

```tsx
import { EmptyState, Button } from "@weiui/react";

<EmptyState
  title="No projects yet"
  description="Create your first project to start collaborating with your team."
  icon={<img src="/illustrations/empty.svg" alt="" />}
  action={<Button>Create project</Button>}
/>
```

### Sizes and orientation

```tsx
<EmptyState size="sm" title="Nothing here" description="Add items to get started." />
<EmptyState size="lg" title="No matches" description="Try different filters." />
<EmptyState
  orientation="horizontal"
  title="Connect a repo"
  description="Import a Git repository to deploy automatically."
  action={<Button>Connect</Button>}
/>
```

### EmptyState props

| Prop           | Type                                  | Default      | Description                                               |
|----------------|---------------------------------------|--------------|-----------------------------------------------------------|
| `title`        | `string`                              | —            | Heading text (required)                                   |
| `description`  | `string`                              | —            | Supporting text                                           |
| `icon`         | `ReactNode`                           | —            | Optional illustration above the title                     |
| `action`       | `ReactNode`                           | —            | Optional call-to-action slot                              |
| `size`         | `"sm" \| "md" \| "lg"`                | `"md"`       | Controls title size, illustration scale, and spacing     |
| `orientation`  | `"vertical" \| "horizontal"`          | `"vertical"` | Horizontal layouts work well inside cards or sidebars    |

### EmptyState accessibility

- `title` renders as an `<h3>` so screen-reader users can navigate the empty state via heading shortcuts.
- Decorative illustrations should be passed as `<img alt="" aria-hidden />`. Let the title carry the meaning.
- The action slot accepts any focusable element — prefer Buttons so the recovery path is obvious in the accessibility tree.
- An empty data region should be wrapped with `role="region"` and an `aria-label` that matches the list it replaces (e.g. `"Projects"`) so screen readers can announce the change.

## When to use which

| Situation                                   | Component                    |
|--------------------------------------------|------------------------------|
| A transient confirmation ("Saved")          | Toast (→ [toast-chip-progress](/docs/components/toast-chip-progress)) |
| An inline, non-blocking message             | **Alert**                    |
| A destructive confirmation                  | Dialog (→ [overlays](/docs/components/overlays)) |
| Short operation in progress                 | **Spinner**                  |
| Measurable multi-step progress              | ProgressBar                  |
| A list has no items                         | **EmptyState**               |

Rule of thumb: if the user can dismiss it and keep working, use Alert. If the feedback is tied to a single in-flight action, use Spinner. If the screen has zero data to show, use EmptyState — never a lone "No results" paragraph.
