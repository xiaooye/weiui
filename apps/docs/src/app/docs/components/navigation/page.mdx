# Navigation

Primitives for structuring movement through your app — in-page tabs, hierarchical breadcrumbs, action menus, links, pagination, top app bars, bottom navigation, and floating action menus. All follow WAI-ARIA Authoring Practices for keyboard navigation.

## Breadcrumb

Hierarchical "you are here" trail. The root renders as `<nav aria-label="Breadcrumb">`. Supports collapse via `<BreadcrumbEllipsis>` when a trail is too deep to fit.

<Preview label="Breadcrumb">
  <BreadcrumbDemo />
</Preview>

```tsx
import {
  Breadcrumb, BreadcrumbItem, BreadcrumbSeparator, BreadcrumbEllipsis,
} from "@weiui/react";

<Breadcrumb>
  <BreadcrumbItem><a href="/">Home</a></BreadcrumbItem>
  <BreadcrumbSeparator />
  <BreadcrumbItem><a href="/products">Products</a></BreadcrumbItem>
  <BreadcrumbSeparator />
  <BreadcrumbItem active>Detail</BreadcrumbItem>
</Breadcrumb>
```

### Collapsed middle with ellipsis

For deep hierarchies, collapse everything except the first and last two segments:

```tsx
<Breadcrumb>
  <BreadcrumbItem><a href="/">Home</a></BreadcrumbItem>
  <BreadcrumbSeparator />
  <BreadcrumbEllipsis />
  <BreadcrumbSeparator />
  <BreadcrumbItem><a href="/products/electronics/audio">Audio</a></BreadcrumbItem>
  <BreadcrumbSeparator />
  <BreadcrumbItem active>Pro Headphones</BreadcrumbItem>
</Breadcrumb>
```

The ellipsis accepts `onClick` and can expand into a Menu showing the collapsed crumbs.

### Breadcrumb props

| Component         | Prop      | Type       | Description                                     |
|-------------------|-----------|------------|-------------------------------------------------|
| `BreadcrumbItem`  | `active`  | `boolean`  | Current page — sets `aria-current="page"`       |

### Breadcrumb accessibility

- Wrapped in `<nav aria-label="Breadcrumb">` for screen-reader discovery.
- Active item carries `aria-current="page"`.
- Separators are `aria-hidden="true"`.
- Ellipsis has `aria-label="Show intermediate crumbs"` when interactive.

## Tabs

In-page content switcher. Supports horizontal + vertical orientation, arrow-key roving focus with optional loop, and controlled / uncontrolled state.

<Preview label="Tabs">
  <TabsDemo />
</Preview>

```tsx
import { Tabs, TabsList, TabsTrigger, TabsContent } from "@weiui/react";

<Tabs defaultValue="overview">
  <TabsList>
    <TabsTrigger value="overview">Overview</TabsTrigger>
    <TabsTrigger value="features">Features</TabsTrigger>
    <TabsTrigger value="pricing">Pricing</TabsTrigger>
  </TabsList>
  <TabsContent value="overview">Overview content</TabsContent>
  <TabsContent value="features">Features content</TabsContent>
  <TabsContent value="pricing">Pricing content</TabsContent>
</Tabs>
```

### Vertical orientation

```tsx
<Tabs defaultValue="profile" orientation="vertical">
  <TabsList orientation="vertical">
    <TabsTrigger value="profile">Profile</TabsTrigger>
    <TabsTrigger value="security">Security</TabsTrigger>
    <TabsTrigger value="billing">Billing</TabsTrigger>
  </TabsList>
  {/* content */}
</Tabs>
```

Arrow-key axis follows orientation: Left/Right for horizontal, Up/Down for vertical.

### Tabs props

| Component     | Prop              | Type                              | Default         | Description                                 |
|---------------|-------------------|-----------------------------------|-----------------|---------------------------------------------|
| `Tabs`        | `defaultValue`    | `string`                          | —               | Uncontrolled initial active tab             |
| `Tabs`        | `value`           | `string`                          | —               | Controlled active tab                       |
| `Tabs`        | `onValueChange`   | `(value: string) => void`         | —               | Fires when the active tab changes           |
| `TabsList`    | `orientation`     | `"horizontal" \| "vertical"`      | `"horizontal"`  | Arrow-key axis                              |
| `TabsList`    | `loop`            | `boolean`                         | `true`          | Arrow wrap at ends                          |

### Tabs keyboard + accessibility

- Left/Right (or Up/Down for vertical): move focus between tabs.
- Home / End: jump to first / last.
- Enter / Space: activate focused tab.
- `role="tablist"`, `role="tab"`, `role="tabpanel"` with `aria-selected`, `aria-controls`, `aria-labelledby` wired.
- Roving `tabindex` — only the active tab is in the Tab sequence.

## Menu

Floating action menu anchored to a trigger. Rendered through a Portal with Floating UI positioning. Supports checkbox and radio items, separators, labels, and keyboard shortcuts.

<Preview label="Menu">
  <MenuDemo />
</Preview>

```tsx
import { Menu, MenuTrigger, MenuContent, MenuItem, MenuSeparator } from "@weiui/react";

<Menu>
  <MenuTrigger className="wui-button wui-button--outline">Actions</MenuTrigger>
  <MenuContent>
    <MenuItem onSelect={() => edit()}>Edit</MenuItem>
    <MenuItem onSelect={() => duplicate()}>Duplicate</MenuItem>
    <MenuSeparator />
    <MenuItem onSelect={() => remove()}>Delete</MenuItem>
  </MenuContent>
</Menu>
```

### Positioning

```tsx
<Menu>
  <MenuTrigger>Open</MenuTrigger>
  <MenuContent side="right" align="start" offset={8}>
    <MenuItem>…</MenuItem>
  </MenuContent>
</Menu>
```

Menu falls back to another side if it collides with the viewport — Floating UI handles flip + shift out of the box.

### Menu props

| Prop        | Type                                                    | Default      | Description                                    |
|-------------|---------------------------------------------------------|--------------|------------------------------------------------|
| `side`      | `"top" \| "right" \| "bottom" \| "left"`                | `"bottom"`   | Preferred placement                            |
| `align`     | `"start" \| "center" \| "end"`                          | `"start"`    | Alignment along the trigger edge               |
| `offset`    | `number`                                                | `4`          | Gap between trigger and content in px          |

### Menu accessibility + keyboard

- `role="menu"` / `role="menuitem"`.
- Arrow keys navigate between items; Home/End jump to extremes.
- Typeahead: press any letter to jump to the next item whose label starts with it.
- Disabled items are skipped.
- Escape closes.
- Content is rendered through `Portal` so it never clips inside scrolling ancestors.

## Pagination

Paged navigation for long lists. Supports controlled current page, configurable sibling + boundary counts, and compact / default variants.

<Preview label="Pagination">
  <PaginationDemo />
</Preview>

```tsx
import { Pagination } from "@weiui/react";

const [page, setPage] = useState(1);

<Pagination page={page} total={40} onChange={setPage} />
```

### Pagination props

| Prop        | Type                         | Default  | Description                                           |
|-------------|------------------------------|----------|-------------------------------------------------------|
| `page`      | `number`                     | —        | Current 1-indexed page                                |
| `total`     | `number`                     | —        | Total number of pages                                 |
| `onChange`  | `(page: number) => void`     | —        | Fires when the user picks a page                      |
| `siblings`  | `number`                     | `1`      | Pages shown on each side of the current page          |
| `boundary`  | `number`                     | `1`      | Pages shown at the first and last boundaries          |

### Pagination accessibility

- Wrapped in `<nav aria-label="Pagination">`.
- Current page carries `aria-current="page"`.
- Prev / Next buttons are `aria-label`-ed and disabled at the boundaries (no focus trap).

## AppBar

Top-of-page chrome with a title, optional toolbar, and sticky behaviour. Renders as `<header role="banner">` when used as the page-level AppBar.

<Preview label="AppBar">
  <AppBarDemo />
</Preview>

```tsx
import { AppBar, Button } from "@weiui/react";

<AppBar sticky>
  <strong>Acme</strong>
  <nav>
    <Button variant="ghost">Docs</Button>
    <Button variant="ghost">Pricing</Button>
  </nav>
  <Button>Sign in</Button>
</AppBar>
```

### AppBar props

| Prop      | Type                                   | Default       | Description                                  |
|-----------|----------------------------------------|---------------|----------------------------------------------|
| `sticky`  | `boolean`                              | `false`       | Position sticky to the top of the viewport   |
| `color`   | `"surface" \| "primary" \| "transparent"` | `"surface"` | Background treatment                          |

## SPA-router integration

All link-accepting components use the native `<a>` underneath. To swap in Next.js `<Link>`, React Router's `<NavLink>`, or TanStack Router's `<Link>`, use the `asChild` pattern where available or render the router link as the child of a `Button variant="link"` / `Link`:

```tsx
import { Link as WuiLink } from "@weiui/react";
import NextLink from "next/link";

{/* Next.js — prefetch + client nav */}
<WuiLink asChild>
  <NextLink href="/docs">Documentation</NextLink>
</WuiLink>

{/* React Router */}
import { NavLink } from "react-router-dom";

<Button variant="link" asChild>
  <NavLink to="/settings">Settings</NavLink>
</Button>
```

The router controls navigation (prefetch, scroll restoration, nested layouts) while WeiUI contributes the visual and a11y contract.

## Responsive patterns

- **Breadcrumb → Menu on narrow viewports:** collapse all but the last crumb into an `<BreadcrumbEllipsis>` that opens a Menu with the rest.
- **Tabs → Select on mobile:** below 480 px, render the same state machine as a `<Select>` — identical keyboard semantics, far better thumb ergonomics.
- **Top nav → Drawer on mobile:** hide the AppBar links behind a Drawer (see [Sidebar & Drawer](/docs/components/sidebar-drawer)).
- **Pagination → "Load more" on mobile:** swap to a single `<Button>` that appends the next page to the list; keeps thumb effort low.

Use container queries (every WeiUI Card supports them) to switch patterns at the right *component* width, not viewport width.
